반면 실시간 스캔 대시보드(그래프·장치 목록)는 Python 서버가 돌아가는 PC가 있어야 합니다. 그 PC에서 실행한 뒤 http://그PC의IP:5000 으로 같은 Wi‑Fi 기기에서 테스트하세요.

여기로 들어왔는데 README만 보이나요? GitHub 저장소 Settings → Pages에서 게시 브랜치가 gh-pages / (root) 인지 확인하세요. main으로 두면 Jekyll이 README만 올립니다.

무엇을 하나요

  • LAN 장치 — 스캔 결과를 대시보드에서 정리합니다.
  • Wi‑Fi — 전용 경로에서 netsh·주변 AP·채널 등 (Windows).
  • 스위치 포트 — IF-MIB SNMP 수집, Cisco는 SNMP 실패 시 SSH 보조. 벤더별 설정은 스위치 포트 매뉴얼.
  • 문서 — API·로그 랩은 Docs에서 확인합니다.

Flask 서버로 대시보드 열기 (Windows)

이 페이지(github.io)에서는 서버를 실행할 수 없습니다. 아래는 프로젝트를 받은 PC에서 할 일입니다. 전체 절차·명령 복사·오류 대응은 매뉴얼을 열어 주세요.

  1. 저장소 클론 또는 ZIP 해제 후 폴더로 이동.
  2. run_dashboard.bat 또는 대시보드 열기.bat 실행 → 제목 NetWork-IP Flask 5000 인 CMD 창이 떠 있어야 합니다.
  3. 브라우저: http://127.0.0.1:5000 · Wi‑Fi 전용 http://127.0.0.1:5000/wifi
  4. (선택) 가상환경·수동 실행: python -m venv .venv → .venv\Scripts\activate → pip install -r requirements.txt → python app.py
  5. 포트 변경: 환경 변수 NETWORK_IP_SEARCH_PORT (예: 8080) 후 python app.py

스위치 장비별 매뉴얼 (포트 UP/DOWN)

대시보드 「스위치 포트 현황」 과 API GET /api/switch/ports 는 devices.yaml 형 인벤토리를 사용합니다.

  • 공통 — SNMP IF-MIB (UDP 161). devices.example.yaml 참고.
  • Cisco — SNMP 우선 · 실패 시 SSH show interfaces status (vendor에 cisco 포함).
  • Juniper / HPE / Aruba — SNMP(IF-MIB) 필수.
  • 유비쿼티(UniFi·에지 등) — 펌웨어별 SNMP 설정 후 관리 IP로 수집 · SSH 보조 없음.
  • 다산 — 제품 SNMP 메뉴에서 community/v3·허용 호스트 설정.
  • Dovado·다보링크 등 SOHO — SNMP·IF-MIB 제공 시만 유효, 미지원 기기는 포트 기능 사용 어려움.
